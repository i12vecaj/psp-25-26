	El primer problema que creo que podría aparecer es que estos hilos estarían accediendo a la variable posiblemente ya modificada por el, o los otros hilos, que hayan accedido primero. d) Compara los modelos de ejecución concurrente, paralela y distribuida, indicando:
Un ejemplo típico de aplicación en cada caso.
Qué tipo de hardware o entorno necesita cada modelo.
Qué ventajas e inconvenientes tiene cada uno en términos de rendimiento y complejidad.
(3 puntos)


	Ejecución concurrente:
		La programación concurrente se puede usar en diversos ámbitos donde se necesite que varios procesos se ejecuten al mismo tiempo, como puede ser en aplicaciones multimedia, 		para reproducir sonido/video mientras se descarga; o en un videojuego, donde los gráficos del entorno tienen que cargar a la vez que las físicas de estos.
		
		La programación concurrente principalmente tira de CPU donde lo que necesita es velocidad de ejecución para que se pueda ejecutar todo bien y al mismo tiempo.
		
		Este tipo de programación tiene un buen rendimiento, ya que si hay 5 procesos que necesitan ejecutarse, este los ejecuta en el tiempo de 1 mismo, me explico, al ejecutarse 		todos los procesos al mismo tiempo, necesita de mucha más CPU, pero también aprovecha mucho mejor el tiempo ya que no se retrasa tanto la ejecución.
		
		
	Ejecución paralela:
		Un ejemplo creo que podría ser un navegador como pueden ser Google, Firefox u Opera, ya que estos pueden llegar a necesitar de varios procesadores en el caso de que tengan 		que trabajar con mucha información, por ejemplo si hay varias pestañas activas abiertas.
		
		Este modelo necesita de un sistema multiprocesador ya que puede llegar a necesitar varias CPU trabajando a la vez para solucionar un mismo problema.
		
		Las principales ventajas de esto es que la ejecución es simultánea y disminuye el tiempo de ejecución de los procesos, también teniendo en cuenta de que gracias al uso de 		varias CPU puede solucionar problemas complejos en una breve cantidad de tiempo. A contraparte, también necesita de compiladores y entornos mas complejos, lo que dificulta 		significativamente la escritura del programa, consume mucha más energía que cualquier otro modelo y se puede llegar a complicar también la comunicación entre archivos o 		procesos.    
		
		
	Ejecución distribuida:
		Un ejemplo típico de este uso es el Cloud Computing de Internet.
		
		Los componentes de este método se interconectan mediante una red, lo que facilita compartir datos y recursos entre sí, tiene mayor flexibilidad y alta disponibilidad. Lo 		malo de este modelo es que la complejidad de programación se incrementa y, si hay algún problema con la red, es probable que haya pérdidas de paquetes de datos, saturación 		de la red al estilo cuello de botella, o similar y diversos problemas de seguridad.
