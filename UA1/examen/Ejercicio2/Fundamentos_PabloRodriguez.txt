a) Explica cómo se comportaría un sistema operativo al ejecutar dos procesos que acceden al mismo archivo de manera simultánea. Indica qué problemas podrían surgir y cómo podrían evitarse. (2 puntos)

- Pues el sistema operativo bloquearía el archivo que los procesos intentaban acceder, lo dejaría en un estado de exclusión. Si esto ocurre puede dar muchos problemas como que los procesos que necesitaban acceder a ese archivo no lo hagan, dando error y otros procesos esperaban a que terminaran darían error haciendo una cadena de errores.
- Para evitarlo hay varias maneras por ejemplo podría darse una prioridad a un proceso en concreto a la hora de acceder a ese archivo o hacer que uno de los procesos que hacen que choquen se ejecute cuando el otro proceso finalice, así irían en orden.

d) Compara los modelos de ejecución concurrente, paralela y distribuida, indicando:

Un ejemplo típico de aplicación en cada caso:
	- La ejecución concurrente se podría usar para ejecuciones mas fáciles y ligeras como un banco donde cada hilo corresponde a un usuario y una cuenta del banco y 	podrían hacer diferentes cosas como ingresar dinero, retirar dinero, hacer transferencias, etc. sin necesidad de esperar a que otro usuario haga su función.

	- La ejecución paralela se puede usar para programas mas pesados que requiera una velocidad de ejecución muy alta, por ejemplo un videojuego
	
	- La ejecución distribuida se puede utilizaría para Google drive o servicios en la nube

Qué tipo de hardware o entorno necesita cada modelo:
	- Para ejecución concurrente se utilizaría un procesadores multinúcleo y entornos mas sencillos
	- Para ejecución paralela se usa sistemas multiprocesador y entornos complejos
	- Para ejecución distribuida el hardware y software están distribuidos y se conectan mediante la red

Qué ventajas e inconvenientes tiene cada uno en términos de rendimiento y complejidad:
	- La ejecución concurrente, al ser concurrente aprovecha mucho mejor la CPU y asi la velocidad de ejecución y no son muy complejos de realizar usando hilos
	
	- La ejecución paralela al usar varias CPU y ejecución simultanea son mejor en rendimiento que la ejecución concurrente y además resuelve problemas complejos, 	el problema es que montar un entorno es mucho mas complicado y es mucho mas difícil de programar de forma paralela.
	
	- La ejecución distribuida tiene buen rendimiento al tener muy buena flexibilidad y disponibilidad pero es complicado por sus problemas de comunicación y 	seguridad al usarse
