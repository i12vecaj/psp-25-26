d)Compara los modelos de ejecución concurrente, paralela y distribuida, indicando:
Un ejemplo típico de aplicación en cada caso.
Qué tipo de hardware o entorno necesita cada modelo.
Qué ventajas e inconvenientes tiene cada uno en términos de rendimiento y complejidad.

Concurrente: Por ejemplo cuando se accede a una base de datos. Varios hilos o procesos pueden acceder a dicha base de datos. En este caso
	     necesitaríamos un gestor de base de datos al que acceder. Al poder acceder varios hilos o procesos es mas rápido, por lo que 
             mejora el rendimiento
Paralela:   Cualquier programa que dependa de otro para ejecutarse. Este tendrá que esperar a que el otro programa se ejecute para 		    ejecutarse.
	    Necesitaría de varios procesadores ya que este modelo empeora el rendimiento y pierde velocidad al tener que esperar para 	  	    ejecutarse.
Distribuida: En este caso es cuando se ejecutan varios procesos de distintos dispositivos. Un ejemplo de uso muy claro seria internet.
	     El principal problema es que si hay muchos procesos en ejecución puede hacer que caiga el hardware. Por otro lado
	     una ventaja seria que casi siempre el recurso estará disponible.
