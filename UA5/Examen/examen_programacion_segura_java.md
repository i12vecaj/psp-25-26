## BLOQUE 1 -- Análisis de código inseguro

Analiza el siguiente código Java:

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

```java
import java.sql.*;
import java.util.Scanner;

public class Login {

    public static void main(String[] args) throws Exception {

        Scanner sc = new Scanner(System.in);

        System.out.println("Usuario:");
        String user = sc.nextLine();

        System.out.println("Password:");
        String pass = sc.nextLine();

        Connection con = DriverManager.getConnection(
                "jdbc:mysql://localhost/app", "root", "root");

        Statement st = con.createStatement();

        String query = "SELECT * FROM usuarios WHERE nombre='"
                + user + "' AND password='" + pass + "'";

        ResultSet rs = st.executeQuery(query);

        if(rs.next()) {
            System.out.println("Acceso concedido");
        } else {
            System.out.println("Acceso denegado");
        }

        con.close();
    }
}
```

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 1.1

Identifica al menos 4 vulnerabilidades o malas prácticas de seguridad en
el código.

Explica brevemente cada una.

Respuesta:

1. El código tiene la consulta SQL directamente con las entradas del usuario (user y pass). Esto permite que un atacante pueda acceder a datos no autorizados o modificar la base de datos.

2. Las credenciales de la base de datos (usuario y contraseña) están hardcodeadas en el código.

3. No se almacenan las contraseñas de forma segura, por lo que se podrán ver las contraseñas.

4. No se manejan adecuadamente los recursos, lo que puede llevar a fugas de recursos si no se cierra correctamente en caso de error.

incluyendo:

- Prevención de SQL Injection\

```java
import java.sql.*;
import java.util.Scanner;
import java.util.Properties;
import java.io.FileInputStream;

public class Login {

    public static void main(String[] args) throws Exception {

        Scanner sc = new Scanner(System.in);

        System.out.println("Usuario:");
        String user = sc.nextLine();

        System.out.println("Password:");
        String pass = sc.nextLine();

        Properties props = new Properties(); // Cargar las credenciales desde un archivo de configuración
        props.load(new FileInputStream("config.properties")); // El archivo config.properties debe tener las propiedades db.url, db.user y db.password

        String url = props.getProperty("db.url");
        String dbUser = props.getProperty("db.user");
        String dbPass = props.getProperty("db.password");

        try (Connection con = DriverManager.getConnection(url, dbUser, dbPass); // Uso de try para asegurar el cierre de la conexión
             PreparedStatement ps = con.prepareStatement(
                     "SELECT * FROM usuarios WHERE nombre=? AND password=?")) {

            ps.setString(1, user);
            ps.setString(2, pass);

            try (ResultSet rs = ps.executeQuery()) { // Uso de try para asegurar el cierre del ResultSet
                if(rs.next()) {
                    System.out.println("Acceso concedido");
                } else {
                    System.out.println("Acceso denegado");
                }
```

### Pregunta 3.1

Explica qué es un certificado digital y para qué se utiliza.

Respuesta:

Un certificado digital es un documento electrónico que vincula una clave pública con una identidad.
Está validado por una Autoridad de Certificación y se utiliza para verificar la identidad y autenticidad en Internet.

### Pregunta 3.3

Explica la diferencia entre:

- KeyStore\
- clave pública\
- clave privada

Respuesta:

- KeyStore: almacena las claves y los certificados digitales de forma segura.
- La clave pública es conocida por todos y se utiliza para cifrar mensajes o verificar firmas digitales.
- La clave privada es conocida por cada interesado y se utiliza para descifrar mensajes cifrados con la clave pública.

## BLOQUE 4 -- Criptografía RSA en Java

Analiza el siguiente código:

```java
KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
keyGen.initialize(1024);

KeyPair pair = keyGen.generateKeyPair();

PublicKey pub = pair.getPublic();
PrivateKey priv = pair.getPrivate();
```

### Pregunta 4.1

¿Qué tipo de criptografía utiliza RSA? Explica brevemente cómo funciona.

Respuesta:

RSA utiliza Criptografía asimétrica.

- El emisor emplea una clave pública que es definida previamente por el receptor
- El receptor emplea la clave privada correspondiente para descrifrar el mensaje.

De esta forma solo el receptor puede desncriptar el mensaje.

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 4.2

Indica qué clave se utiliza para:

a) Cifrar un mensaje confidencial\
b) Firmar digitalmente

Respuesta:

a) Para cifrar un mensaje confidencial se utilizaría la clave pública del receptor, ya que solo el receptor con la clave privada podría descifrar el mensaje.
b) Para firmar digitalmente se utilizaría la clave privada del emisor, ya que cualquier receptor con la clave pública del emisor podrá verificar la firma digital.

### Pregunta 4.4

Indica si esta configuración es segura y justifica:

```java
keyGen.initialize(512);
```

Respuesta:

No, esta configuración no es segura. La clave contiene pocos bits por lo que puede ser descubierta con técnicas de fuerza bruta.
Se recomendaría utilizar una clave con más bits para garantizar una mayor seguridad, como por ejemplo de 1024 o 2048 bits.
