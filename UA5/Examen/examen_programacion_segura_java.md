``` java

import java.util.Scanner;

public class Login {

    public static void main(String[] args) throws Exception {

        Scanner sc = new Scanner(System.in);

        System.out.println("Usuario:");

            System.out.println("Acceso concedido");
```

## BLOQUE 2 -- Conceptos de programación segura

Responde a las siguientes preguntas:

### Pregunta 2.1

Explica qué es una vulnerabilidad de tipo SQL Injection y cómo se
previene en Java.

- SQL Injection es una vulnerabilidad de seguridad que ocurre cuando un atacante puede insertar o manipular consultas SQL a través de la entrada de datos de una aplicación. Esto puede permitir al atacante acceder, modificar o eliminar datos en la base de datos, e incluso ejecutar comandos administrativos.

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 2.2

¿Por qué es una mala práctica almacenar contraseñas en texto plano?\
¿Qué técnica se debe usar en su lugar?

- Es una mala práctica almacenar contraseñas en texto plano porque si alguien accede a esa información, tendría acceso a todas las contraseñas sin ningún tipo de protección, lo que sería de gran peligro para la información delicada de una empresa.
En su lugar se deberían de usar técnicas de hashing que lo que hace es cifrar la contraseña de tal manera que no se pueda recueperar la contraseña que había anteriormente mediante el hash.

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 2.3

¿Qué es el principio de mínimo privilegio? Pon un ejemplo en Java o en
bases de datos.

- El principio de mínimo privilegio es un concepto de seguridad que establece que cada usuario, programa o proceso debe tener solo los privilegios necesarios para realizar sus tareas específicas y nada más. Esto ayuda a reducir el riesgo de daños accidentales o malintencionados al limitar el acceso a recursos críticos.

- Un ejemplo en una base de datos podría ser el hecho de que a mi ahora que me voy de prácticas solo me dieran permisos en la BBDD para poder hacer consultas, que es lo que hice en las practicas del año pasado, asi puedo ayudar a sacar información importante pero no puedo modificar nada ni eliminarlo.

### Pregunta 4.2

Indica qué clave se utiliza para:

a)  Cifrar un mensaje confidencial\

- Para cifrar un mensaje confidencial se suele usar la clave pública, ya que esta es la que define el receptor para encriptarla y esa clave pública sera empleada por el emisor.

b)  Firmar digitalmente

 - Para firmar de manera digital se suele usar la estructura JCA, que permite cambiar implementaciones de algoritmos sin tocar
 código, luego para el proceso de firma, inicializamos el objeto Signature en modo firma con la clave privada y posteriormente usamos el método update para pasar los datos firmados y luego finalmente el método sign, lo que hace que se firme con la clave privada, luego para verificar esa firma se usa el mismo proceso pero con la clave pública, lo que hace que se verifique con la clave pública y si es correcta se da por válida esa firma digital
