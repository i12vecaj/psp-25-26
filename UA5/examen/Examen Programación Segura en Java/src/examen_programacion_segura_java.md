## BLOQUE 1 -- Análisis de código inseguro

Analiza el siguiente código Java:

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

``` java
import java.sql.*;
import java.util.Scanner;

public class Login {

    public static void main(String[] args) throws Exception {

        Scanner sc = new Scanner(System.in);

        System.out.println("Usuario:");
        String user = sc.nextLine();

        System.out.println("Password:");
        String pass = sc.nextLine();
        
        String uri  = System.getenv("DB_URI");
        String dbUser = System.getenv("DB_USER");
        String dbPass = System.getenv("DB_PASS");
        
        String sql = "SELECT * FROM usuarios WHERE nombre=? AND password = ?";
        
        try (Connection con = DriverManager.getConnection(uri, dbUser, dbPass);
             PreparedStatement ps = con.prepareStatement(sql)) {
             
            ps.setString(1, user);
            ps.setString(2, pass);
            
            try (ResultSet rs = ps.executeQuery()) {
                if(rs.next()) {
                    System.out.println("Acceso concedido");
                } else {
                    System.out.println("Acceso denegado");
                }
            } catch (SQLException e) {
                System.err.println("Error al ejecutar la consulta: " + e.getMessage());
            }
        }

        Connection con = DriverManager.getConnection(
                "jdbc:mysql://localhost/app", "root", "root");

        Statement st = con.createStatement();

        String query = "SELECT * FROM usuarios WHERE nombre='"
                + user + "' AND password='" + pass + "'";

        ResultSet rs = st.executeQuery(query);

        if(rs.next()) {
            System.out.println("Acceso concedido");
        } else {
            System.out.println("Acceso denegado");
        }

        con.close();
    }
}
```

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 1.1

Identifica al menos 4 vulnerabilidades o malas prácticas de seguridad en
el código.

Explica brevemente cada una.

1. Tener las credenciales de la base de datos (usuario y contraseña) hardcodeadas en el código fuente, lo que puede ser fácilmente descubierto por un atacante.
2. Hacer la consulta SQL concatenando directamente los datos de entrada del usuario, lo que hace que el código sea vulnerable a ataques de SQL Injection.
3. No proteger o encriptar las credenciales en la base de datos, lo que puede llevar a que un atacante que acceda a la BBDD pueda obtener las contraseñas de los usuarios.
4. No controlar de manera adecuada los recursos, como la conexión a la BBDD, lo que puede provocar fugas de recursos si no se cierran correctamente, por ejemplo, en caso de error.

## BLOQUE 2 -- Conceptos de programación segura

Responde a las siguientes preguntas:

### Pregunta 2.1

Explica qué es una vulnerabilidad de tipo SQL Injection y cómo se
previene en Java.

- Una vulnerabilidad de tipo SQL Injection ocurre cuando un atacante puede insertar o "inyectar", como el propio nombre indica, código malicioso en una consulta SQL a través de la entrada de datos del usuario.
Esto puede permitir al atacante manipular la consulta SQL para ver, modificar o eliminar datos de manera no autorizada. 

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 2.2

¿Por qué es una mala práctica almacenar contraseñas en texto plano?\
¿Qué técnica se debe usar en su lugar?

- Es una mala práctica porque si un atacante obtiene acceso a dicho archivo o BBDD puede usar tales datos de forma libre y sin restricciones, lo que puede llevar a la suplantación de identidad del usuario, acceso no autorizado a sistemas privados de dicho usuario y otros ataques relacionados.
- Se deben almacenar las contraseñas usando métodos de hash seguros, para así estar encriptadas y evitar que sean legibles para cualquiera en caso de que un atacante acceda a la BBDD o archivo donde se almacenan.

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 2.3

¿Qué es el principio de mínimo privilegio? Pon un ejemplo en Java o en
bases de datos.

- Es un concepto de seguridad en el cual cada usuario o programa debe tener solo los privilegios necesarios para realizar únicamente sus tareas, nada más. Esto ayuda a limitar el posible daño total que un atacante puede realizar si compromete una cuenta de usuario o programa. Por ejemplo, en una BBDD, un usuario que solo necesita leer datos no debería tener permisos de escritura o eliminación.



/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 2.4

Indica si las siguientes afirmaciones son verdaderas o falsas y
justifica:

a) PreparedStatement evita SQL Injection
- Verdadero. Es una clase de Java que permite ejecutar consultar SQL de manera segura y eficiente, ya que precompila la consulta y luego asigna cada parámetro de forma segura, evitando la posibilidad de que haya código malicioso.

b) RSA es criptografía simétrica
- Falso. Es un algoritmo de criptografía asimétrica, lo que significa que utiliza 2 claves, una pública y otra privada, la pública para cifrar datos y la privada para descifrarlos, lo que permite una comunicación segura sin necesidad de compartir ninguna clave.

c) La clave privada puede compartirse públicamente
- Falso. Si esto sucede se está comprometiendo la seguridad total del sistema, ya que cualquier persona puede acceder a él y filtrar posibles datos importantes.

d) Un certificado digital contiene una clave pública
- Verdadero. El sistema verifica la autenticidad de dicha clave y establecer una conexión segura entre las dos partes.

### Pregunta 3.1

Explica qué es un certificado digital y para qué se utiliza.

- Es un documento electrónico que vincula una clave pública con la indentidad de una persona u organización, y se utiliza para verificar la autenticidad de dicha clave y establecer una conexión segura entre las partes que se comunican.

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 3.2

¿Qué contiene un certificado digital?

Menciona al menos 3 elementos.

1. La clave pública de la persona titular.
2. La identidad del titular, que incluye datos personales del mismo.
3. La firma digital del titular, lo cual garantiza la autenticidad del certificado.
4. La fecha de emisión y de expiración del mismo.

/\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*/

### Pregunta 3.3

Explica la diferencia entre:

-   KeyStore: Es una clase de Java utilizada para almacenar y gestionar claves y certificados de manera segura.
-   clave pública: Es una parte de un sistema de criptografía asimétrica y sirve para cifrar datos o verificar firmas digitales.
-   clave privada: Es la otra parte de un sistema de criptografía asimétrica y se utiliza para descifrar datos cifrados con la clave pública o para crear firmas digitales.

## BLOQUE 4 -- Criptografía RSA en Java

Analiza el siguiente código:

``` java
KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
keyGen.initialize(1024);

KeyPair pair = keyGen.generateKeyPair();

PublicKey pub = pair.getPublic();
PrivateKey priv = pair.getPrivate();
```

### Pregunta 4.1

¿Qué tipo de criptografía utiliza RSA? Explica brevemente cómo funciona.

- Es criptografía asimétrica, significa que usa dos claves, una pública que encripta los datos, y una privada que los desencripta.
